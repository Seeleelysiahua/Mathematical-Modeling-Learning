## 1.2 适用范围

本文档适用于以下场景：
- 航空公司或物流企业的航线规划

- 需要严格记录迭代过程的学术或工程计算

## 1.3 术语与符号释义

| 术语/符号 | 释义 |
|-----------|------|
| **∞（无穷大）** | 表示两城市之间无直达航路，在程序中用充分大的正数表示（如Integer.MAX_VALUE或Double.POSITIVE_INFINITY） |
| **单源最短路径** | 从一个指定的源点出发，计算到图中所有其他节点的最短距离 |
| **邻接矩阵** | 一个n×n的方阵，第i行第j列的元素表示节点cᵢ到节点cⱼ的直接距离/票价 |
| **Dijkstra算法** | 由荷兰计算机科学家Edsger Dijkstra提出的贪心算法，用于求解带权有向图或无向图的单源最短路径 |
| **永久集合（S集）** | 算法中已确定最短距离且不再更新的节点集合 |
| **临时集合（T集）** | 算法中尚未确定最短距离的节点集合 |
| **路径回溯** | 根据前驱节点信息，从目标节点反向追踪至源点，得到完整路径 |

## 1.4 输入数据说明

### 1.4.1 邻接矩阵原始数据

```
直达票价邻接矩阵 C（6×6对称矩阵）：

           c₁   c₂   c₃   c₄   c₅   c₆
    c₁ [   0,   50,   ∞,   40,   25,   10 ]
    c₂ [  50,    0,   15,   20,   ∞,   25 ]
    c₃ [   ∞,   15,    0,   10,   20,   ∞ ]
    c₄ [  40,   20,   10,    0,  100,   25 ]
    c₅ [  25,    ∞,   20,  100,    0,   55 ]
    c₆ [  10,   25,    ∞,   25,   55,    0 ]
```

### 1.4.2 矩阵解读规则

1. **对角线元素**：C[i][i] = 0，表示城市到自身的距离为0
2. **对称性**：C[i][j] = C[j][i]，本问题为无向图
3. **直达票价**：C[i][j]（i≠j）表示从cᵢ到cⱼ的直达票价
4. **无穷大（∞）**：表示两城市之间没有直达航路，需要中转

### 1.4.3 ∞的处理规则

| 处理环节 | 处理方式 |
|----------|----------|
| **程序存储** | 使用Integer.MAX_VALUE（2147483647）或Double.POSITIVE_INFINITY |
| **加法运算** | 任何数与∞相加结果仍为∞ |
| **比较运算** | 任何有限数与∞比较时，有限数小于∞ |
| **路径输出** | 最终结果若仍为∞，表示两城市不可达 |

# 第三章 算法执行规范

## 3.1 算法参数设定

| 参数 | 值 |
|------|-----|
| 源点 | c₁ |
| 目标城市集合 | {c₂, c₃, c₄, c₅, c₆} |
| 节点总数 | n = 6 |
| 初始距离数组 | dist[1..6]，dist[1]=0，其余为∞ |

## 3.2 迭代轮次记录格式

每一轮迭代必须记录以下内容：

```
═══════════════════════════════════════════════════════════════
第 k 轮迭代（k = 1, 2, ..., 5）
═══════════════════════════════════════════════════════════════

【本轮选定节点】
  从临时集合T中选出距离最小的节点：c_x
  选定距离：dist[c_x] = d

【永久集合S更新】
  S_old = { ... }
  S_new = { ..., c_x }

【距离更新判定】
  对于每个节点c_j (j ∈ T_new)：
    若 dist[c_x] + C[x][j] < dist[c_j]：
      更新：dist[c_j] = dist[c_x] + C[x][j]
      记录前驱：prev[c_j] = c_x

【更新后的临时距离表】
  ┌─────────┬────────┬────────┬────────┬────────┬────────┐
  │  节点   │  c₁    │  c₂    │  c₃    │  c₄    │  c₅    │  c₆  │
  ├─────────┼────────┼────────┼────────┼────────┼────────┤
  │ dist    │  ✓0    │  50    │  ∞     │  40    │  25    │  10   │
  │ prev    │   -    │  c₁    │   -    │  c₁    │  c₁    │  c₁   │
  │ 状态    │  S     │  T     │  T     │  T     │  T     │  T    │
  └─────────┴────────┴────────┴────────┴────────┴────────┘

【本轮变化说明】
  - 新增永久节点：c_x（距离d）
  - 更新的临时节点：[列出更新的节点及新距离]
  - 未变化的节点：[列出未更新的节点]
```

## 3.3 永久最短节点集合更新规则

1. **初始状态**：S = {c₁}（源点直接加入永久集合）
2. **每轮操作**：
   - 从T中选出dist值最小的节点c_min
   - 将c_min从T移至S
   - 更新所有邻居节点的临时距离
3. **终止条件**：当|S| = n时，算法结束

## 3.4 节点临时距离更新判定逻辑

对于当前选中的节点c_u及其邻居c_v：

```
IF dist[u] + C[u][v] < dist[v] THEN
    dist[v] = dist[u] + C[u][v]
    prev[v] = u
END IF
```

**关键约束**：
- 仅当新路径更短时才更新
- 仅考虑尚未加入永久集合的节点
- ∞参与运算时保持∞的特性

# 第五章 结果验收标准

## 5.1 路线正确性核验方法

### 5.1.1 总票价验证

对于每个目标城市c_t，执行以下核验：

```
验证公式：∑(路径上所有边的权重) = dist[c_t]

示例验证（假设路径为 c₁→c₆→c₄→c₃）：
  C[1][6] + C[6][4] + C[4][3] = dist[3]
  10 + 25 + 10 = 45
  ✓ 验证通过
```

### 5.1.2 路径存在性验证

- 验证路径上相邻节点之间C[i][j] ≠ ∞
- 验证路径的首节点为c₁，末节点为目标城市

### 5.1.3 最优性验证

- 验证不存在其他路径的总票价更小
- 可通过穷举法或与理论最优值比对

## 5.2 算法全步骤可复现判定要求

### 5.2.1 复现性检查清单

| 检查项 | 要求 |
|--------|------|
| 初始状态 | dist[1]=0, dist[2..6]=∞, S={c₁} |
| 轮次数量 | 共5轮（6个节点-1） |
| 每轮选择 | 选出T中dist最小的节点 |
| 距离更新 | 仅当新路径更短时更新 |
| 最终结果 | dist[i]为从c₁到cᵢ的最短距离 |

### 5.2.2 中间结果一致性

- 任意两轮之间的状态转换必须符合算法规则
- 不允许跳过步骤或遗漏更新

# 附录A：算法伪代码

```
DIJKSTRA(G, source):
    // G = (V, E) 为带权图，source 为源点
    // dist[v] 存储从 source 到 v 的最短距离
    // prev[v] 存储 v 的前驱节点

    INITIALIZE(G, source):
        for each vertex v in V:
            dist[v] = INFINITY
            prev[v] = UNDEFINED
        dist[source] = 0
        S = { source}     // 永久集合
        T = V - {source}  // 临时集合

    while T is not empty:
        // 从临时集合中选出距离最小的节点
        u = vertex in T with minimum dist[u]

        // 将 u 移入永久集合
        remove u from T
        add u to S

        // 更新 u 的邻居节点的临时距离
        for each neighbor v of u:
            if dist[u] + weight(u, v) < dist[v]:
                dist[v] = dist[u] + weight(u, v)
                prev[v] = u

    return dist[], prev[]
```
